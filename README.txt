## 與ninite的差別
ninite 將應用程式安裝在system級別, 通常會修改註冊表, 需要root權限

## 與protable apps的差別
PA 透過一些特殊方式將應用程式打包成單一執行檔.


Brickv 的重點不再於Portable或自動下載安裝
而在於他提供一個虛擬開發環境, 可以方便的選擇要載入的APP, 而不用擔心

1. 自動安裝依賴
2. 可選擇版本和架構
3. 跨平台Windows, Linux, Mac
4. 切換APP
5. 不需要root權限
6. 可以任意更動或是Patch該App, 而不用擔心影響到其他專案
7. 透過環境變數來處理APP之間的依賴與APP設定檔
